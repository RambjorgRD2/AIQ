**2. A client-side `<script>` tag (NOT `is:inline`) BEFORE the closing `</body>` (but after `<slot />`):**

```javascript
import { supabase } from '../lib/supabase.ts'

// Subscribe to auth state changes — fires on page load and on sign in/out
supabase.auth.onAuthStateChange((_event, session) => {
  const user = session?.user ?? null

  // Toggle auth UI elements
  document.querySelectorAll('[data-auth-show]').forEach(el => {
    ;(el as HTMLElement).style.display = user ? '' : 'none'
  })
  document.querySelectorAll('[data-auth-hide]').forEach(el => {
    ;(el as HTMLElement).style.display = user ? 'none' : ''
  })

  // Show user email
  const emailEl = document.getElementById('auth-user-email')
  if (emailEl && user) {
    emailEl.textContent = user.email ?? ''
  }

  // Expose user globally for page-level scripts
  ;(window as any).__supabaseUser = user
})

// Handle sign-out button(s)
document.querySelectorAll('[data-sign-out]').forEach(btn => {
  btn.addEventListener('click', async () => {
    await supabase.auth.signOut()
    window.location.href = '/AIQ/'
  })
})
```

**Key details:**
- The `data-auth-show` elements start with `style="display:none"` (hidden by default) — the `onAuthStateChange` callback sets `display: ''` (restoring default) when a session exists
- The `data-auth-hide` elements start with `style="display:block"` — the callback hides them when signed in
- `window.__supabaseUser` is set to the Supabase `User` object (or null) for any page script to consume
- `signOut()` uses the default scope ('global') — terminates the session on all devices
- After sign-out, redirect to `/AIQ/` (the landing page)
