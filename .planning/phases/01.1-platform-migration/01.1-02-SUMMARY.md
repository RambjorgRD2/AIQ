## Decisions Made

- Used `vercel pull / vercel build / vercel deploy --prebuilt` three-step pattern (not single `vercel deploy`) — this is the recommended GitHub Actions pattern; `vercel pull` downloads Vercel project config and any env vars set in dashboard before build, ensuring environment is consistent with Vercel dashboard configuration

- `VERCEL_ORG_ID` and `VERCEL_PROJECT_ID` set at job `env:` level (not individual step) — required by Vercel CLI for authentication across all steps
