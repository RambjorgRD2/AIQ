<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Supabase tables via SQL Editor</name>
  <what-to-do>
    Open the Supabase dashboard, go to SQL Editor, and run the following SQL. This creates the three tables with RLS policies.

    ```sql
    -- lesson_progress
    create table if not exists public.lesson_progress (
      id uuid primary key default gen_random_uuid(),
      user_id uuid not null references auth.users(id) on delete cascade,
      lesson_id text not null,
      course_id text not null,
      completed_at timestamptz not null default now(),
      unique(user_id, lesson_id)
    );
    alter table public.lesson_progress enable row level security;
    create policy "Users can manage own lesson progress"
      on public.lesson_progress for all
      using (auth.uid() = user_id)
      with check (auth.uid() = user_id);

    -- assessment_results
    create table if not exists public.assessment_results (
      id uuid primary key default gen_random_uuid(),
      user_id uuid not null references auth.users(id) on delete cascade,
      score integer not null,
      tier text not null,
      tier_label text not null,
      radar_data jsonb,
      completed_at timestamptz not null default now()
    );
    alter table public.assessment_results enable row level security;
    create policy "Users can manage own assessment results"
      on public.assessment_results for all
      using (auth.uid() = user_id)
      with check (auth.uid() = user_id);

    -- reflection_notes
    create table if not exists public.reflection_notes (
      id uuid primary key default gen_random_uuid(),
      user_id uuid not null references auth.users(id) on delete cascade,
      lesson_id text not null,
      note_text text not null default '',
      updated_at timestamptz not null default now(),
      unique(user_id, lesson_id)
    );
    alter table public.reflection_notes enable row level security;
    create policy "Users can manage own reflection notes"
      on public.reflection_notes for all
      using (auth.uid() = user_id)
      with check (auth.uid() = user_id);
    ```
  </what-to-do>
  <resume-signal>Type "tables created" once the SQL has run successfully in the Supabase dashboard</resume-signal>

  <behavior>
    - upsertLessonProgress({ userId, lessonId, courseId }): upserts to lesson_progress with onConflict on (user_id, lesson_id); returns void; silently no-ops if not authenticated
    - upsertAssessmentResult({ userId, score, tier, tierLabel, radarData }): inserts to assessment_results (no unique constraint — each attempt is a new row); returns void
    - upsertReflectionNote({ userId, lessonId, noteText }): upserts to reflection_notes with onConflict on (user_id, lesson_id); returns void
    - getCompletedLessons(userId, courseId): selects lesson_id from lesson_progress where user_id = userId and course_id = courseId; returns string[]
    - getLatestAssessmentResult(userId): selects most recent row from assessment_results ordered by completed_at desc limit 1; returns result object or null
    - getReflectionNote(userId, lessonId): selects note_text from reflection_notes for the given user+lesson; returns string or null
    - All functions use the supabase singleton from src/lib/supabase.ts
    - All functions throw/log on Supabase error rather than swallowing silently
