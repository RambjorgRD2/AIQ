# Phase 1.2 Context â€” localStorage â†’ Supabase Sync

## Domain
Authenticated users have their lesson progress, assessment results, and reflection notes persisted to Supabase. On sign-in, existing localStorage data is migrated. On every new completion/result event, data is written to Supabase in real time.

## Decisions

### Supabase Schema
- Three tables: `lesson_progress` (user_id, lesson_id, course_id, completed_at), `assessment_results` (user_id, score, tier, radar_data, completed_at), `reflection_notes` (user_id, lesson_id, note_text, updated_at)
- Row Level Security (RLS) on all tables: users can only read/write their own rows
- `lesson_id` = the lesson slug (e.g. `"context-windows"`) â€” already used as URL param

### Sync Strategy
- **On sign-in**: read existing localStorage keys, batch-upsert to Supabase, then keep localStorage as cache (don't delete it â€” keeps things working if offline)
- **On new events** (lesson complete, assessment complete, reflection save): write to Supabase immediately if authenticated; write to localStorage always (so unauthenticated users still work)
- **Read path**: on page load, if authenticated, prefer Supabase data over localStorage for rendering completion state and results
